Imagine you have a function Sort that sorts an array. The function uses another function to compare the elements of the array. Functors allow you to pass this comparison function into your Sort function as an argument. Functors have many applications in Computer Science. For starters, you will use functors to control how your lists are sorted. 

 

How Functors are Used?

Here, a test function (this could be a member function belonging to a class) calls the Sort function and passes it the Bigger and then the Smaller functors to let the Sort function know how it wants the elements of an array compared to each other. Note that Bigger and Smaller are classes and as such must be supplied any template arguments they require. Not so with SortArray function which does not. 

 

 

 

What is a Functor?

A functor is a simple class or a structure whose parenthesis operator has been overloaded:

Here the Bigger and Smaller classes are templated and each have their parenthesis operator overloaded to take two arguments and return a bool. This enables the two classes to take any type that supports the less than operator

Note: Both Smaller and Bigger classes use the less than operator so that the provider of the T object (type) need not implement both less than and greater than operators

Receiving the functor as an argument:

In the sort function, we have a template argument called compare and the function itself has an argument of type compare named cmp. The template argument allows objects of various types of classes (types) to be passed to the function.  

The function uses the cmp argument to compare the elements of the array. Notice that the object of the class Compare is being called like a function because its parenthesis argument has been overloaded.

The function needs a Compare template argument because we are sending an object from one of two different classes (Smaller or Bigger) as the third argument of the Sort function.
